About The Project

Melbourne Housing Price Predictor

This project uses machine learning algorithms to predict house prices in Melbourne based on various property features and location data.

Our model is trained on data from thousands of Melbourne properties and can predict prices with approximately 80% accuracy. It takes into account factors like location, property size, number of rooms, and many other features.

Our Machine Learning Models

We compared several models to find the best one for predicting Melbourne housing prices

Linear Regression

A simple baseline model that establishes linear relationships between features and house prices.

  • Simple and interpretable
  • Establishes baseline accuracy
  • Limited with non-linear relationships

Random Forest

An ensemble model that combines multiple decision trees to make predictions with high accuracy.

  • High accuracy (80%+)
  • Handles non-linear relationships
  • Provides feature importance
Best Performer

Gradient Boosting

A sequential ensemble technique that builds trees one at a time, each fixing errors of the previous ones.

  • High accuracy
  • Reduces bias in predictions
  • More complex to tune

Key Features We Analyze

Our model considers numerous property attributes to make accurate predictions

Location Data

Suburb, distance from CBD, region, and council area significantly impact property values.

Property Attributes

Number of rooms, bathrooms, bedrooms, car spaces, and property type.

Size Metrics

Land size and other dimensional attributes help determine property value.

Sale Attributes

Method of sale, seller/agent information, and date of transaction.

Neighborhood Data

Property count in the area and other neighborhood characteristics.

Geospatial Features

Precise latitude and longitude coordinates for exact location analysis.

Our Technical Process

1 Data Collection & Cleaning

We collected data on over 13,500 Melbourne properties and performed thorough data cleaning, handling missing values and removing outliers.

2 Feature Engineering

We processed categorical variables using one-hot encoding and extracted meaningful features from the raw data.

3 Model Training & Selection

We trained and compared multiple models including Linear Regression, Random Forest, and Gradient Boosting, selecting the best performer.

4 Evaluation & Optimization

Models were evaluated using metrics like RMSE and R² score, with hyperparameter tuning performed to optimize performance.

5 Deployment

The best performing model was saved and deployed to this web application, allowing users to make real-time predictions.
